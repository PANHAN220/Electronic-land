# Smart Electronic Development Assessment System (Smart eDA)

**Smart eDA** เป็นระบบบริการออนไลน์ของรัฐบาลควีนส์แลนด์  ที่ออกแบบมาเพื่อช่วยในการจัดเตรียม ยื่น และประเมินคำขอพัฒนาที่ดินอย่างมีประสิทธิภาพ  โดยมุ่งเน้นการให้บริการที่ใช้งานง่าย สำหรับทั้งผู้สมัครและเจ้าหน้าที่ 

## 🌐 เกี่ยวกับโปรเจกต์ (About the Project)
โปรเจกต์นี้เป็นการพัฒนาระบบ Multi-page Web Interface สำหรับการจัดการคำขอพัฒนาที่ดิน โดยจำลองขั้นตอนการทำงานตั้งแต่การรับคำร้อง การตรวจสอบข้อมูลภูมิศาสตร์จากหน่วยงานต่าง ๆ ไปจนถึงการอนุมัติขั้นสุดท้าย 

## 🛠️ กระบวนการทำงานของระบบ (Workflow)
อ้างอิงจากขั้นตอนการดำเนินงานในกรณีศึกษา:

### 1. การยื่นคำร้องและตรวจสอบเบื้องต้น (Initial Submission & Validation)
* **การรับคำร้อง**: เริ่มต้นกระบวนการเมื่อได้รับคำขอจากผู้สมัคร 
* **การตรวจสอบพื้นที่**: ดึงข้อมูลภูมิศาสตร์จาก Cadastre และส่งให้ City Council ตรวจสอบความถูกต้องเบื้องต้น 
* **การเสนอราคา**: หากแผนงานผ่านการตรวจสอบ ระบบจะส่งใบเสนอราคา (Quote) ตามประเภทของโครงการ (ที่อยู่อาศัยหรือพาณิชย์) 

### 2. การประเมินโดยละเอียด (Detailed Assessment Phase)
* **ตรวจสอบข้อขัดแย้ง (DMR)**: ตรวจสอบกับกรมทางหลวงเพื่อป้องกันการทับซ้อนกับโครงการถนนในอนาคต 
* **การขอใบอนุญาตเฉพาะทาง**:
    * **NRW**: ขอใบอนุญาตดัดแปลงที่ดินหากมีการเปลี่ยนแปลงสภาพแวดล้อมทางธรรมชาติ 
    * **EPA**: ขอใบอนุญาตด้านสิ่งแวดล้อมหากพื้นที่อยู่ในเขตคุ้มครองตามกฎหมาย 

### 3. การสิ้นสุดและการติดตาม (Finalization & Tracking)
* **การแจ้งผล**: แจ้งการอนุมัติขั้นสุดท้ายเมื่อได้รับใบอนุญาตครบถ้วน 
* **การติดตามสถานะ**: ผู้สมัครสามารถติดตามความคืบหน้า (Track Progress) ได้ตลอดเวลา 
* **การยกเลิก**: ผู้สมัครสามารถยกเลิกคำร้องได้ทุกเมื่อ ซึ่งระบบจะทำการแจ้งเตือนทุกฝ่ายและเพิกถอนใบอนุญาตที่เกี่ยวข้อง 

## 💻 ข้อกำหนดทางเทคนิค (Technical Requirements)
เว็บไซต์นี้พัฒนาขึ้นภายใต้มาตรฐานทางเทคนิคดังนี้:
* **Framework**: Vue.js สำหรับการสร้าง Web Application 
* **Styling**: Tailwind CSS สำหรับการจัดการเลย์เอาต์และ Responsive Design
* **Navigation**: Vue Router สำหรับระบบ Multi-page
* **Structure**: ออกแบบในลักษณะ Component-based เพื่อความง่ายในการนำกลับมาใช้ใหม่ 
* **Design Standards**: รองรับการใช้งานทั้ง Desktop, Tablet, และ Mobile รวมถึงการนำหลัก Usability Heuristics มาปรับใช้เพื่อความง่ายในการเข้าถึงข้อมูล 

## 📋 ข้อมูลเพิ่มเติม (Additional Features)
* รองรับโหมดการแสดงผลแบบ **Dark Mode** 
* มีการใช้ **Design Tokens** และ **Tailwind CSS Utility Classes** เพื่อความสม่ำเสมอของดีไซน์
